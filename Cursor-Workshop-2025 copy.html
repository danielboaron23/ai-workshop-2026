Avoid These

Common Mistakes

Learn from others' errors

‚ùå Asking for too much at once

"Build me a complete e-commerce site with cart, checkout, user accounts..."

‚úì Fix: Break into small, specific tasks. One feature at a time.

‚ùå Not reading the output

Accepting all changes without reviewing what AI actually did.

‚úì Fix: Always review the diff before accepting changes.

‚ùå Forgetting to commit

Working for hours without saving progress, then something breaks.

‚úì Fix: Commit after each working feature. Small, frequent commits.

‚ùå Ignoring errors

Continuing to add features while existing code is broken.

‚úì Fix: Always fix errors before moving to the next task.

‚ùå Not providing context

"Fix the button" ‚Äî which button? What's wrong with it?

‚úì Fix: Use @file references and be specific about location.

‚ùå Fighting the AI

Repeatedly asking the same thing when it's not working.

‚úì Fix: Rephrase your request. Try a different approach.

Introduction

What Are Sub-Agents?

AI specialists that work independently on specific tasks

Sub-agents are specialized AI assistants that handle specific types of tasks. Each runs in its own context window with custom prompts, specific tool access, and independent permissions.

Key Benefits
  • ‚úì Preserve context ‚Äî Isolate exploration from main conversation
  • ‚úì Enforce constraints ‚Äî Limit tools a subagent can use
  • ‚úì Run in parallel ‚Äî Multiple subagents work simultaneously
  • ‚úì Control costs ‚Äî Route tasks to faster, cheaper models

When to Use Sub-Agents

Match the right approach to the task

‚úì Use Sub-Agents When
  • ‚úì Task produces verbose output you don't need
  • ‚úì You want to enforce specific tool restrictions
  • ‚úì Work is self-contained and can return a summary
  • ‚úì High-volume operations (tests, docs, logs)
  • ‚úì Parallel research with independent investigations
  • ‚úì Multi-step workflows that can be chained
‚úó Use Main Conversation When
  • ‚úó Task needs frequent back-and-forth
  • ‚úó Multiple phases share significant context
  • ‚úó Making a quick, targeted change
  • ‚úó Latency matters (subagents start fresh)
  • ‚úó Iterative refinement needed
