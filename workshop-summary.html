# Figma Make Prompt Generator

## Overview
This skill helps create effective prompts for
Figma Make by applying the 3C's methodology.

## The 3C's Framework

### Clarity
Be specific about what you want:
- Screen type (login, dashboard, settings)
- Key elements to include
- Primary user action

### Context
Provide relevant background:
- App type/industry
- Target platform (mobile, web, tablet)
- Design style (minimal, playful, corporate)

### Constraints
Define boundaries:
- Screen dimensions
- Color limitations
- Must-have vs nice-to-have elements

## Prompt Template

When generating a prompt, use this structure:

```
Create a [screen type] for a [app type] app.

Include:
- [Key element 1]
- [Key element 2]
- [Key element 3]

Style: [Design style description]
Platform: [Target platform, dimensions]
```

## Examples

**User request:** "Login page for fitness app"

**Generated prompt:**
"Create a login screen for a fitness tracking
app. Include email/password fields, social
login buttons (Google, Apple), 'Forgot password'
link, and a motivational fitness image or
illustration. Use energetic colors (orange
accents) with a clean white background.
Mobile-first, 375px width."

# Figma Make Guidelines

## Workflow Best Practices

### Before Starting
1. Define the end goal clearly
2. Break complex designs into steps
3. Gather reference images if needed
4. Know your constraints (size, style)

### During Creation
1. Start simple, add complexity gradually
2. One change at a time - easier to iterate
3. Use Point & Edit for quick fixes
4. Save versions before major changes

### Common Pitfalls to Avoid
- DON'T: Ask for everything in one prompt
- DON'T: Use vague descriptions
- DON'T: Ignore platform constraints
- DON'T: Skip the planning phase

### Do's
- DO: Be specific about elements
- DO: Mention colors, sizes, spacing
- DO: Provide context (app type, audience)
- DO: Iterate and refine

## Troubleshooting

### Result doesn't match expectations?
1. Check if prompt was specific enough
2. Add more constraints
3. Try breaking into smaller steps
4. Use reference images

### Performance issues?
1. Simplify the design first
2. Work on sections separately
3. Reduce number of elements

## Output Quality Tips

- Request specific dimensions
- Mention accessibility needs
- Specify interactive states
- Include micro-copy examples
